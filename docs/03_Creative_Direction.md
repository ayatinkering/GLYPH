# 03 — Creative Direction

> Designing a computational instrument for movement.

Version 1.0

---

# Design Objective

GLYPH should not feel like software.

It should feel like discovering a scientific instrument.

Every screen should communicate calmness, curiosity and craft.

The application should disappear behind the artwork.

Users should remember the Mandala Commit—not the interface that created it.

The design should feel timeless.

Five years from now it should still look modern.

---

# Emotional Response

Every interaction should evoke one or more of these emotions.

Wonder.

Calm.

Curiosity.

Craft.

Presence.

Reflection.

Never urgency.

Never productivity.

Never competition.

Never dopamine.

The user should slow down.

Not speed up.

# Visual Philosophy

Whitespace is a feature.

Silence is a feature.

Typography is a feature.

Motion is a feature.

The interface exists only to frame the computational artwork.

Nothing should compete with the Mandala.

---

# Layout Philosophy

Every screen follows one rule.

One primary object.

Landing

↓

Hero statement.

Walking

↓

Mandala.

Commit

↓

Artwork.

Gallery

↓

Collection.

Never multiple competing focal points.

---

# Grid

Use an 8pt spacing system.

Maximum content width

1200px desktop.

720px reading width.

Mobile first.

Extremely generous margins.

Allow breathing room.

---

# Whitespace

If something feels crowded,

remove something.

Never shrink the spacing.

The interface should feel effortless.

Almost empty.

---

# Typography

Not color.

Not boxes.

Not shadows.

# Motion Language

Everything should breathe.

Nothing should bounce.

Animation vocabulary

Fade

Scale

Bloom

Rotate

Blur

Opacity

Duration

300–800ms

Easing

ease-in-out

Motion should feel biological.

Like plants growing.

Not software animating.

---

# The Mandala

The Mandala is not decoration.

It is the interface.

Everything else supports it.

During a walk

the Mandala grows.

When the walk ends

the Mandala settles.

When exported

the Mandala becomes permanent.

---

# Illustration Style

No illustrations.

No stock photos.

No hero graphics.

The Mandala itself is the brand illustration.

---

# Icons

Lucide.

Thin strokes.

Minimal.

Never filled.

Never colorful.

---

# Buttons

Rounded.

Large.

Soft.

Text first.

No gradients.

No shadows stronger than 4px blur.

Buttons should almost disappear.

---

# Cards

Subtle radius.

White background.

---

# Export Philosophy

Every exported artifact should be printable.

If someone hangs it on a wall,

it should still feel beautiful.

Think

museum poster

botanical print

engineering blueprint

rather than

Instagram graphic.

---

# Landing Page

The homepage should feel almost empty.

Hero.

One sentence.

One animation.

One button.

Nothing more.

Users should understand the product within five seconds.

---

# Walking Screen

Almost no interface.

The artwork fills the experience.

The UI disappears.

Walking should feel meditative.

---

# Commit Screen

This is the emotional climax.

The user has returned.

The Mandala blooms into its final form.

Metadata fades in.

The beat softly loops.

The Commit number appears.

Only then do the action buttons become visible.

Download.

Commit.

Share.

Nothing interrupts the reveal.

---

# Design Rule

Whenever faced with two design choices,

choose the quieter one.

Whenever faced with two animations,

choose the slower one.

Whenever faced with two layouts,

choose the one with more whitespace.

Whenever faced with two features,

choose the simpler one.

The Mandala should always remain the loudest voice in the room.

---

# Closing Principle

GLYPH should feel less like opening an application

and more like opening

a beautifully printed scientific journal documenting a mathematical phenomenon that just happened to be created by your own footsteps.
